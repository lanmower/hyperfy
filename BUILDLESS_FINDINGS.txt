Why bundling happens:
1. Combine 65 JSX files into 2 chunks
2. Transform JSX to React.createElement
3. Tree-shake unused code
4. Generate source maps
5. Hash bundle names for cache-busting

Issues with current pipeline:
- 2-5 second recompilation on every change
- All 65 modules re-bundled even if 1 file changed
- Source maps bloat output
- Esbuild process consumes memory

WHAT BUILDLESS REMOVES
- esbuild.context() - No longer needed
- Bundle hashing - Use query string instead
- Tree-shaking pass - Direct loading is naturally "shaken"
- Source map generation - Browser dev tools work with source
- Module resolution - Browser ESM loader handles it

WHAT BUILDLESS KEEPS
- Asset copying (public files, WASM)
- Environment configuration
- HTML template rendering

CONCLUSION: Esbuild only needed for optional production optimization
