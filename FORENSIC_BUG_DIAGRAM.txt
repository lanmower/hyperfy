CRITICAL BUG #4: SERVER TIME BASE MISMATCH
────────────────────────────────────────────
Time Synchronization Flow:
  Server: serverTime = 1000ms
    │
    ├─ ClientNetwork.setServerTime(1000)
    │   └─ serverTimeOffset = moment.now() - 1000  ← USES moment!
    │
  Later: SnapshotProcessor receives snapshot
    │
    ├─ serverTimeOffset = performance.now() - 1000  ← USES performance!
    │
  getTime() called:
    │
    └─ return moment.now() - serverTimeOffset  ← USES moment!

  PROBLEM:
  ├─ moment.now() = ~1.7 billion ms (epoch-based)
  ├─ performance.now() = ~0-600,000 ms (relative to page load)
  ├─ TWO DIFFERENT TIME BASES!
  │
  EFFECT:
  ├─ Clock skew accumulates
  ├─ Animations jitter (wrong delta times)
  ├─ Network state diverges
  ├─ Extended play: 10ms+ drift per minute
  └─ BLOCKS DEPLOYMENT ✗

Network Packet Queue:
  Queue: [
    [method1, data1] ✓

    [method3, data3] ✓
  ]
    │
    ├─ flush()
    │   ├─ Process: [method1, data1] ✓
    │   ├─ Skip: [null, data2] ← SILENT!
    │   │         (no logging, no error, just continue)
    │   └─ Process: [method3, data3] ✓
    │
  EFFECT:
  ├─ Network data lost silently
  ├─ Entity state becomes inconsistent
  ├─ No indication to developer or player
  └─ Makes debugging impossible


HIGH BUG #6: SNAPSHOTCODEC DOUBLE TIMESTAMP
──────────────────────────────────────────────
Timestamp Calculation:
  T0: encode() called
    ├─ serverTime = performance.now() = 1000ms
    └─ Packet created

  T0 + 100ms: Packet in transit

  T0 + 100ms: decode() called
    ├─ serverTimeOffset = 1000 - performance.now()
    │                   = 1000 - 1100 = -100ms ← WRONG!
    └─ Should be +100ms (network latency)

  EFFECT:
  ├─ Time offset off by 200ms
  ├─ Animations timing wrong
  ├─ Network state updates arrive at wrong time

MEDIUM BUG #8: ZLIB MODULE LOADING RACE
────────────────────────────────────────
Module Load Timeline:
  T0: Compressor.js loads
    ├─ require('zlib') starts
    └─ hasZlib = false (initially)

  T0 + 1ms: Compressor instance created
    └─ compress() called immediately

  T0 + 1ms: compress() checks hasZlib
    ├─ hasZlib still false! ← Race condition!
    └─ returns {compressed: false}

  T0 + 50ms: require('zlib') finishes
    └─ hasZlib = true (too late!)

  EFFECT:
  ├─ First packets sent uncompressed
  ├─ Bandwidth wasted
  ├─ Performance degraded
