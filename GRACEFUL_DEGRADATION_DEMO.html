1. Feature Detection

The system automatically detects browser capabilities at startup:

// Detect all features const featureDetector = new FeatureDetector() const features = await featureDetector.detect() const capabilities = featureDetector.getCapabilities() // Results: { canRender3D: true, // WebGL/WebGL2 available canUseAudio: true, // Web Audio API available canUseVoiceChat: true, // WebRTC + Microphone canUseVideoChat: true, // WebRTC + Camera canUseWebSocket: true, // WebSocket available canUseStorage: true, // IndexedDB/LocalStorage canUseXR: false, // WebXR not available canUseGamepad: true, // Gamepad API available }

Debug Console Access:

// In browser console: window.__DEBUG__.getFeatures() // All detected features window.__DEBUG__.getCapabilities() // Computed capabilities window.__DEBUG__.getDegradationStatus() // Current degradation state

2. Asset Fallback System

When assets fail to load, fallbacks are automatically provided:

Supported Asset Types:

Models (.glb)
→ Gray box placeholder
Textures (.png/.jpg)
→ White 64x64 texture
Audio (.mp3/.wav)
→ Silent audio buffer
Scripts (.js)
→ No-op script
HDR Environments
→ Neutral gray HDR
Avatars (.vrm)
→ Gray capsule mesh
Emotes
→ Empty animation
Videos
→ Black texture

Example - Missing Model:

[ClientLoader] Error loading file (asset://missing-model.glb): 404 Not Found [FALLBACK] Using model fallback for asset://missing-model.glb: 404 Not Found ✓ Gray box placeholder rendered instead ✓ Application continues without crash

Fallback Log Access:

// In browser console: const log = window.__DEBUG__.getFallbackLog() // Example output: [ { type: "model", url: "asset://missing-model.glb", error: "404 Not Found", timestamp: "2025-12-27T10:30:45.123Z" }, { type: "texture", url: "asset://broken-texture.png", error: "Failed to decode image", timestamp: "2025-12-27T10:30:47.456Z" } ]

3. Component Degradation

Systems gracefully disable when required features are unavailable:

Audio System (ClientAudio.js):

// If Web Audio API unavailable: [ClientAudio] Web Audio API unavailable - audio disabled ✓ System continues in silent mode ✓ No audio crashes ✓ All audio methods safely return early // Check status: world.audio.degraded // true if audio unavailable

LiveKit Voice Chat (ClientLiveKit.js):

// If WebRTC/Microphone unavailable: [ClientLiveKit] Voice chat unavailable - audio/video disabled ✓ Voice chat UI disabled ✓ Application remains functional ✓ Text chat still available // Check status: world.livekit.degraded // true if voice unavailable world.livekit.status.available // false

Network System (ClientNetwork.js):

// If WebSocket unavailable or connection lost: [ClientNetwork] WebSocket not available - entering offline mode [ClientNetwork] Offline mode activated: Connection closed ✓ Switches to readonly mode ✓ Last known state cached ✓ User notified via chat message // Chat notification shown: "Connection lost. Entering offline mode. Your changes will not be saved." // On reconnection: "Connection restored. Syncing with server..." ✓ Full snapshot requested ✓ Stale entities cleared // Check status: world.network.offlineMode // true if offline

7. Architecture Summary

Initialization Flow:

1. FeatureDetector.detect() └─ Detects all browser capabilities └─ Returns features object 2. world.capabilities = featureDetector.getCapabilities() └─ Stores computed capabilities on world └─ Available to all systems 3. System initialization (ClientAudio, ClientLiveKit, etc.) └─ Check world.capabilities └─ Set degraded flag if feature unavailable └─ Continue initialization or skip gracefully 4. Asset loading (ClientLoader) └─ Attempt to load asset └─ On failure → FallbackManager.getFallback() └─ Log fallback usage └─ Return fallback asset

Key Design Principles:

  • Fail gracefully: Never crash, always provide fallback
  • Transparent logging: All fallbacks logged with [FALLBACK] prefix
  • User-friendly messages: Clear explanations, not stack traces
  • Feature flags: degraded/offlineMode flags for runtime checks
  • Debug access: All state accessible via window.__DEBUG__
  • Automatic recovery: Reconnect on connection restore
